1. The Problem the Paper Solves

Before this paper, neural machine translation used a basic encoder–decoder:

  • The encoder (an RNN) reads the whole source sentence and squeezes it into one fixed vector c.
  • The decoder (another RNN) generates the translation, using only that single vector.
The bottleneck: short and long sentences must both fit in the same fixed-size vector. Translation quality drops sharply when sentences exceed about 20 words.

Analogy: imagine reading a 30-word English sentence, writing a 10-word summary, hiding the original, and only translating from your summary. You will lose details — especially for long inputs.

2. The Core Idea: Attention

Instead of compressing the source sentence into one vector, let the decoder look back at all source words every time it generates a new target word, and let it decide which source words matter most for the current step.

Human analogy: a translator does not memorize the English sentence and write French with eyes closed. They keep glancing back at specific English words while writing each French word. Attention gives the model the same ability.

3. The Five Key Equations

Equation 1 — Alignment Score

eij = a(si−1, hj)

A scalar number that says: "how important is source word j for producing target word i?"

  • i — the target step (the word we are about to generate)
  • j — the source position
  • si−1 — the decoder's previous hidden state
  • hj — the encoder's annotation for source word j
  • a(·) — a small neural network (explained in section 4)

Equation 2 — Softmax to Probabilities

αij = exp(eij) / ∑k=1..Tx exp(eik)

Convert all scores at step i into probabilities that sum to 1. These are the attention weights.

Equation 3 — Context Vector

ci = ∑j=1..Tx αij · hj

A weighted average of all source annotations, where higher-weight words contribute more. Notice ci is different at every decoder step.

Equation 4 — Update the Decoder State

si = f(si−1, yi−1, ci)

f is a GRU (or LSTM). It mixes:

  • si−1 — what the decoder remembered before
  • yi−1 — the previously generated word
  • ci — the part of the source it just attended to

Equation 5 — Output the Next Word

p(yi | y<i, x) = g(yi−1, si, ci)

A softmax over the target vocabulary. The model picks the most probable next word.

The Whole Pipeline

(si−1, hj)→ eij→ softmax→ αij→ weighted sum→ ci→ update si→ predict yi

4. Inside the Alignment Network a(s, h)

The function a(·) is the most important detail of the paper. It is a tiny one-hidden-layer neural network:

eij = vaT · tanh( Wa · si−1 + Ua · hj )

What each parameter is

ParameterRoleShape
Wa Projects the decoder state into a shared comparison space (n', n)
Ua Projects each source annotation into the same space (n', 2m)
va Collapses the combined vector to a single scalar score (n',)

Where n = decoder hidden size, 2m = annotation size (twice m because the encoder is bidirectional), and n' = alignment hidden size.

Important: these are learned by backpropagation, exactly like every other neural-net weight. No one hand-codes them. The model figures out alignment from translation loss alone.

Why split into two matrices?

Mathematically tanh(W·[s; h]) would also work. The split into Wa·s + Ua·h is an efficiency trick:

  • Wa·si−1 — same for all j → compute once per decoder step.
  • Ua·hj — same for all i → compute once per source position, before decoding starts.

6. What is the Decoder State si?

Many learners get confused here. The decoder state is just the LSTM/GRU's memory: a vector that summarizes everything the decoder needs to remember up to step i.

Example update with concrete numbers:

Inputs:
  s_1 = [0.2, -0.5, 0.7]   # memory after writing "أنا"
  y_1 = "أنا"               # last produced word
  c_2 = [0.77, 0.02, 0.29] # context (mostly "love")

Output:
  s_2 = [0.6, -0.2, 0.5]   # new memory after writing "أنا" and looking at "love"
The numbers in s2 are not directly meaningful. They are a learned vector representation. The model uses them internally to:
  • Remember what it has produced so far ("أنا").
  • Remember what part of the source it just looked at (the context "love").
  • Carry that information forward to predict the next word ("أحب").
Think of si as a notebook: it gets updated each step with everything the decoder needs to know to make the next decision.

7. Full Paper Coverage

§1 Introduction

The fixed-size context vector is the bottleneck. The proposal: model jointly learns to align and translate — each output word is conditioned on a different context vector.

§2 Background: Encoder–Decoder

Encoder produces a single c = hT. Decoder uses the same c for every output. This is the design they replace.

§3 Learning to Align and Translate

  • 3.1 Decoder: output depends on ci (different per step), not c.
  • 3.2 Encoder: bidirectional RNN. Each hj = [→hj; ←hj] summarizes the whole sentence but is biased toward word j.

§4 Experiment Setup

  • Dataset: WMT'14 English→French.
  • Vocabulary: 30k most frequent words; rest mapped to <UNK>.
  • Architectures: encoder/decoder hidden size 1000, alignment hidden 1000, GRU units, deep-output maxout layer.
  • Trained four models: RNNencdec-30, RNNencdec-50, RNNsearch-30, RNNsearch-50.
  • ~5 days training each, Adadelta, batch size 80, beam search at decode time.

Hand-crafted alignment in IBM/HMM models, Schwenk's neural reranking, encoder-decoder predecessors (Kalchbrenner & Blunsom, Sutskever, Cho), and Graves's handwriting synthesis (which inspired soft attention).

Replacing the fixed vector with a dynamic, soft alignment lets NMT scale to long sentences and approach phrase-based SMT quality — a major milestone for neural MT in 2014.

8. Applying Bahdanau Attention to Flickr8k Image Captioning

This paper inspired Xu et al. 2015 "Show, Attend and Tell", which is exactly Bahdanau attention applied to images. The mapping is straightforward:

Bahdanau (translation)Captioning (Show, Attend & Tell)
Source sentence (T words)Image (H×W spatial grid)
Encoder = BiRNNEncoder = CNN (ResNet-50, VGG-19)
Annotations h1..hTRegion features a1..aL
Each hj ∈ ℝ2mEach ai ∈ ℝD
Target sentence (translation)Caption (English words)
Soft alignment over source wordsSoft alignment over image regions

Concrete sizes for Flickr8k

  • Encoder: ResNet-50 pretrained on ImageNet. Strip the final FC + avgpool.
  • Output feature map: 7×7×2048.
  • Flatten spatial dims → L = 49 region features, each of size D = 2048.
  • Decoder: single-layer LSTM, hidden size 512.
  • Word embedding size: 256, vocab ~5k–10k.
  • Attention hidden size: 512.

The full architecture diagram

Image
  ↓
ResNet-50 (frozen)
  ↓
feature map 7x7x2048
  ↓ flatten
{a_1, a_2, ..., a_49}, each ∈ ℝ^2048
  ↓
││ LSTM Decoder + Bahdanau Attention ││
  At each step t:
    e_{t,i} = v_a^T tanh(W_a · h_{t-1} + U_a · a_i)
    α_{t,i} = softmax(e_{t,i})
    z_t     = Σ_i α_{t,i} · a_i
    h_t     = LSTM(h_{t-1}, [E·y_{t-1}; z_t])
    p(y_t)  = softmax(MLP(h_t, z_t, E·y_{t-1}))
  ↓
Caption: "a dog is running on the grass"

The alignment equation, restated for images

et,i = vaT · tanh( Wa · ht−1 + Ua · ai )

Same equation, the only changes are:

  • ht−1 — LSTM hidden state (size 512)
  • ai — image feature at region i (size 2048)
  • Shapes: Wa = (512, 512), Ua = (512, 2048), va = (512,)

Initialization trick from Show, Attend and Tell

h_0 = tanh(W_init_h · mean(a_i))
c_0 = tanh(W_init_c · mean(a_i))   # for LSTM cell state

Initialize the LSTM with the mean image feature instead of zeros.

Doubly-stochastic regularization

Encourages every region to be looked at across the caption (not just one):

L = −Σt log p(yt) + λ · Σi ( 1 − Σt αt,i )2

Use λ = 1.0 as in the paper.

9. PyTorch Skeleton

import torch
import torch.nn as nn
import torchvision.models as models

class EncoderCNN(nn.Module):
    def __init__(self):
        super().__init__()
        resnet = models.resnet50(pretrained=True)
        # remove avgpool and fc; keep feature map
        modules = list(resnet.children())[:-2]
        self.cnn = nn.Sequential(*modules)
        for p in self.cnn.parameters():
            p.requires_grad = False  # freeze (unfreeze later if you want)

    def forward(self, images):
        # images: (B, 3, 224, 224)
        feats = self.cnn(images)               # (B, 2048, 7, 7)
        feats = feats.permute(0, 2, 3, 1)      # (B, 7, 7, 2048)
        feats = feats.view(feats.size(0), -1, 2048)  # (B, 49, 2048)
        return feats

class BahdanauAttention(nn.Module):
    def __init__(self, encoder_dim=2048, decoder_dim=512, attn_dim=512):
        super().__init__()
        self.W_a = nn.Linear(decoder_dim, attn_dim, bias=False)
        self.U_a = nn.Linear(encoder_dim, attn_dim, bias=False)
        self.v_a = nn.Linear(attn_dim, 1, bias=False)

    def forward(self, features, hidden):
        # features: (B, 49, 2048)   hidden: (B, 512)
        Uh = self.U_a(features)                # (B, 49, 512)
        Ws = self.W_a(hidden).unsqueeze(1)     # (B, 1,  512)
        scores = self.v_a(torch.tanh(Uh + Ws)).squeeze(-1)  # (B, 49)
        alpha = torch.softmax(scores, dim=1)   # (B, 49)
        context = (alpha.unsqueeze(-1) * features).sum(dim=1)  # (B, 2048)
        return context, alpha

class CaptionDecoder(nn.Module):
    def __init__(self, vocab_size, embed_dim=256, decoder_dim=512, encoder_dim=2048):
        super().__init__()
        self.embedding = nn.Embedding(vocab_size, embed_dim)
        self.attention = BahdanauAttention(encoder_dim, decoder_dim)
        self.lstm = nn.LSTMCell(embed_dim + encoder_dim, decoder_dim)
        self.init_h = nn.Linear(encoder_dim, decoder_dim)
        self.init_c = nn.Linear(encoder_dim, decoder_dim)
        self.fc = nn.Linear(decoder_dim, vocab_size)

    def init_hidden(self, features):
        mean_feat = features.mean(dim=1)               # (B, 2048)
        h = torch.tanh(self.init_h(mean_feat))         # (B, 512)
        c = torch.tanh(self.init_c(mean_feat))
        return h, c

    def forward(self, features, captions):
        B, T = captions.shape
        h, c = self.init_hidden(features)
        embeds = self.embedding(captions)              # (B, T, 256)
        outputs = []
        alphas = []
        for t in range(T):
            context, alpha = self.attention(features, h)
            lstm_input = torch.cat([embeds[:, t, :], context], dim=1)
            h, c = self.lstm(lstm_input, (h, c))
            outputs.append(self.fc(h))
            alphas.append(alpha)
        return torch.stack(outputs, dim=1), torch.stack(alphas, dim=1)
Loss: standard cross-entropy on next-token prediction + doubly-stochastic regularization on the stacked alphas. Mask <pad> tokens out of the loss.

10. Practical Tips for the Final Project

Dataset Prep

  • Flickr8k: 8000 images × 5 captions = 40k pairs.
  • Lowercase, strip punctuation, add <start> / <end>.
  • Build vocab from training split only (avoid leakage).
  • Cap captions at 20 tokens for speed.

Training

  • Optimizer: Adam, lr=4e-4 (decoder), lr=1e-4 (CNN if fine-tuning).
  • Teacher forcing during training.
  • Freeze CNN for first 5 epochs, then optionally unfreeze last block.
  • Batch size 32–64 depending on GPU.

Inference

  • Greedy decoding for fast checks; beam search (beam=3 or 5) for final results.
  • Save alpha tensors so you can plot heatmaps.

Evaluation

  • BLEU-1 / BLEU-4 (use NLTK or pycocoevalcap).
  • METEOR, CIDEr if you have time.
  • Human-readable samples are more convincing than numbers in a final-project demo.

Visualization — Your Killer Demo

  • For each generated word, reshape alpha from (49,) back to (7, 7).
  • Upsample to image size with bilinear interpolation.
  • Overlay as a translucent heatmap on the original image.
  • You can literally show: "when the model said 'dog', it looked here."
  • Combine with Grad-CAM for a complementary gradient-based view.

Common Pitfalls

  • Forgetting ImageNet normalization on input images.
  • Not masking <pad> in loss → model learns to output padding.
  • Skipping the doubly-stochastic regularization → attention collapses to one region.
  • Not sorting batches by length when packing sequences.

Summary in One Sentence

Attention = let the decoder, at every step, compute a custom-weighted summary of the input, where the weights are learned and depend on what the decoder is currently trying to produce.

This single idea grew into the Transformer (2017), and into every modern LLM. In your project, that same idea will let your captioning model "look at" different image regions as it writes each word — and let you visualize where it looked.
